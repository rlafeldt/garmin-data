Create `src/biointelligence/analysis/client.py` with:

1. `get_anthropic_client(settings: Settings) -> anthropic.Anthropic` -- factory creating client from settings.anthropic_api_key.

2. `analyze_prompt(client, prompt: AssembledPrompt, model: str, max_tokens: int = 4096, temperature: float = 0.3) -> tuple[DailyProtocol, dict]` -- implements TWO layers of retry:

   **Layer 1 -- Transport retry (tenacity decorator):** Wrap the function with tenacity retry (stop_after_attempt(3), wait_exponential(multiplier=2, min=4, max=60), retry on anthropic.RateLimitError, anthropic.InternalServerError, anthropic.APIConnectionError). This handles transient API transport errors only.

   **Layer 2 -- Parse failure retry (explicit loop inside the function, per user decision):** After calling `client.messages.parse(...)`, the SDK raises `pydantic.ValidationError` if Claude's response does not conform to the DailyProtocol schema. Handle this with an explicit retry loop (up to 3 attempts):
   - Wrap the `client.messages.parse(...)` call in a for loop with `max_parse_attempts = 3`.
   - On `pydantic.ValidationError`: log the raw response content (the text Claude returned) at ERROR level for debugging using `log.error("parse_failure", attempt=attempt, raw_response=response.content, error=str(e))`. Continue to next attempt.
   - If all 3 parse attempts fail, re-raise the last ValidationError with a clear message.
   - On success, break out of the loop.

   This satisfies the locked user decision: "Parse failure: retry up to 3 times on invalid JSON, then fail gracefully and log raw response for debugging."

   After successful parse: check stop_reason -- raises ValueError on "refusal", logs warning on "max_tokens". Extracts response.parsed_output as protocol. Returns (protocol, metadata_dict) where metadata has model, input_tokens, output_tokens, stop_reason. Logs token usage via structlog.

   Note: With `messages.parse()`, the raw API call itself may succeed (valid HTTP response) but Pydantic validation of the parsed output can fail. The tenacity decorator does NOT catch ValidationError -- that is intentional. Transport errors and parse errors are separate concerns with separate retry strategies.
