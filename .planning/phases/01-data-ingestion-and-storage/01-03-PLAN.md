Fix two failing tests in `tests/test_client.py` class `TestSettings` by preventing pydantic-settings from reading the real `.env` file during test execution.

**Root cause:** `Settings()` is configured with `env_file=".env"`. When a real `.env` exists, file-sourced values override env vars that were `delenv`'d (because those vars were never in the process env to begin with -- they were in the file). pydantic-settings v2 supports a `_env_file` constructor parameter to override this per-instance.

**Fix test_settings_has_defaults (lines 37-48):**
1. Add `monkeypatch.delenv('TARGET_TIMEZONE', raising=False)` to clear any env var for optional fields being tested
2. Add `monkeypatch.delenv('GARMIN_TOKEN_DIR', raising=False)` for the same reason
3. Add `monkeypatch.delenv('LOG_JSON', raising=False)` for the same reason
4. Change `Settings()` to `Settings(_env_file=None)` to disable .env file loading
5. The test should then see the code defaults: `garmin_token_dir="~/.garminconnect"`, `target_timezone="Europe/Berlin"`, `log_json=False`

**Fix test_settings_validates_required_fields (lines 50-59):**
1. Change `Settings()` to `Settings(_env_file=None)` to prevent .env from supplying required fields
2. Keep the existing `monkeypatch.delenv` calls for the 4 required fields
3. With both process env vars cleared AND .env file disabled, pydantic-settings should raise `ValidationError` for missing required fields

**Also update test_settings_loads_from_env_vars (lines 17-36):**
1. Change `Settings()` to `Settings(_env_file=None)` to ensure the test verifies env var loading only (not .env file loading), making it deterministic regardless of .env file presence

**Do NOT modify:**
- `src/biointelligence/config.py` -- the production code is correct
- Any tests in `TestGetAuthenticatedClient` -- those already work correctly
- Any other test files
