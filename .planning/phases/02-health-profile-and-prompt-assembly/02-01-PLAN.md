  <action>
    1. Create `src/biointelligence/trends/__init__.py` exporting TrendDirection, MetricTrend, TrendResult, compute_trends.

    2. Create `src/biointelligence/trends/models.py`:
       - TrendDirection(str, Enum): IMPROVING, DECLINING, STABLE, INSUFFICIENT = "insufficient_data"
       - MetricTrend(BaseModel): avg: float | None = None, min_val: float | None = None, max_val: float | None = None, direction: TrendDirection = TrendDirection.INSUFFICIENT
       - TrendResult(BaseModel): window_start: date, window_end: date, data_points: int, metrics: dict[str, MetricTrend]
       - TRENDED_METRICS dict mapping metric name -> {"lower_is_better": bool}: hrv_overnight_avg (False), resting_hr (True), sleep_score (False), total_sleep_seconds (False), body_battery_morning (False), avg_stress_level (True), training_load_7d (False)

    3. Create `src/biointelligence/trends/compute.py`:
       - TREND_FIELDS: comma-separated string of date + 7 trended metric names + is_no_wear for the Supabase SELECT
       - fetch_trend_window(client: Client, target_date: date, window_days: int = 7) -> list[dict]: Query Supabase daily_metrics with .select(TREND_FIELDS).gte("date", start.isoformat()).lt("date", target_date.isoformat()).eq("is_no_wear", False).order("date", desc=False). Use tenacity retry decorator (same pattern as storage/supabase.py).
       - compute_direction(values: list[float], lower_is_better: bool = False, min_data_points: int = 4, threshold: float = 0.05) -> TrendDirection: Filter None from values. If len < min_data_points return INSUFFICIENT. Split-half comparison: mean(first_half) vs mean(second_half). If first_half mean is 0, return STABLE. Compute pct_change = (second - first) / abs(first). If lower_is_better, invert pct_change sign. Return IMPROVING if > threshold, DECLINING if < -threshold, else STABLE.
       - compute_trends(client: Client, target_date: date) -> TrendResult: Call fetch_trend_window, then for each metric in TRENDED_METRICS: extract values (skipping None), compute avg/min/max via statistics.mean/min/max, compute direction. Return TrendResult.

       Log with structlog: trend window dates, data points found, computed directions.

    4. Create `tests/test_trends.py`:

       - Test no-wear exclusion (the query itself filters; verify the .eq("is_no_wear", False) call)
       - Test empty response returns all INSUFFICIENT
