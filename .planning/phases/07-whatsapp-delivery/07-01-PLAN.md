    WhatsApp Renderer (test_whatsapp_renderer.py):
    - render_whatsapp(protocol, target_date) returns a string
    - Output starts with "*Daily Protocol*" header line with formatted date
    - Second line contains "Readiness: *{score}/10*"
    - When protocol.alerts is non-empty, alert blocks appear before domain sections, each with "*[SEVERITY] title*", description, and "Action: suggested_action"
    - Domain sections appear in order: Sleep, Recovery, Training, Nutrition, Supplementation -- each with emoji header and *bold* section name
    - Sleep section contains *Quality:* and *Architecture:* bold keys with plain text values
    - Recovery section contains *Status:*, *HRV:*, *Body Battery:* bold keys
    - Training section contains *Intensity:*, *Type:*, *Duration:* bold keys
    - Nutrition section contains *Calories:*, *Macros:*, *Hydration:* bold keys
    - Supplementation section lists adjustments as "- item" lines with *Timing:* key
    - Each domain includes 1-2 sentence trimmed reasoning (plain text, no bold)
    - Message ends with "*Why This Matters*" section containing overall_summary
    - When protocol.alerts is empty, no alert section appears
    - Output does not exceed 32,768 characters (guard with truncation/warning if over)

    WhatsApp Sender (test_whatsapp_sender.py):
    - send_whatsapp(body_text, target_date, settings) returns DeliveryResult
    - On success: calls httpx.post to https://graph.facebook.com/v21.0/{phone_number_id}/messages with correct headers (Authorization: Bearer {token}, Content-Type: application/json) and template payload (messaging_product: whatsapp, to: recipient, type: template, template.name: daily_protocol, template.language.code: en, template.components with body parameter)
    - On success: returns DeliveryResult(success=True, email_id=message_id_from_response)
    - On transient failure (429, 500, 502, 503, 504, TransportError): retries up to 3 times with exponential backoff
    - On permanent failure (401, 400): fails immediately without retry, returns DeliveryResult(success=False, error=...)
    - On all retries exhausted: returns DeliveryResult(success=False, error=...) -- does not raise

    Settings (test_whatsapp_sender.py::TestSettingsWhatsApp):
    - Settings loads WHATSAPP_ACCESS_TOKEN, WHATSAPP_PHONE_NUMBER_ID, WHATSAPP_RECIPIENT_PHONE from env
    - All three default to empty string "" when not set
  </behavior>
