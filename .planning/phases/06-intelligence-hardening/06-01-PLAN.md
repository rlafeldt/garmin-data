   - Constants: `EXTREME_OUTLIER_WARNING = 2.5`, `EXTREME_OUTLIER_CRITICAL = 3.0`

For "body_battery_drain" special case: compute (body_battery_max - body_battery_min) from each row. Treats None as non-anomalous.
   - `_make_outlier_alert(metric_name: str, current_value: float, z_score: float, trend: MetricTrend) -> Alert`: creates Alert with WARNING if abs(z) >= 2.5, CRITICAL if abs(z) >= 3.0. Uses lower_is_better from TRENDED_METRICS to determine direction context. pattern_name = "single_metric_outlier".
   - `_check_pattern(pattern: ConvergencePattern, trends_28d: TrendResult, trend_rows: list[dict]) -> bool`: for each MetricCheck in pattern, calls _check_consecutive_days. Returns True only if ALL metric checks pass.
   - `_make_convergence_alert(pattern: ConvergencePattern) -> Alert`: creates Alert from pattern fields.
   - `detect_anomalies(today_metrics: DailyMetrics, trends_28d: TrendResult, trend_rows: list[dict]) -> AnomalyResult`: orchestrates single-metric outlier checks (iterate TRENDED_METRICS, skip None/INSUFFICIENT, compute z-score from today's value vs 28d baseline) and convergence pattern checks (iterate CONVERGENCE_PATTERNS). Returns AnomalyResult with combined alerts. Log via structlog: detected alert count, metrics checked.
