```python
class DeliveryResult(BaseModel):
    """Result of a protocol delivery attempt."""
    date: date
    email_id: str | None = None
    success: bool
    error: str | None = None
```

`send_email(html: str, text: str, subject: str, target_date: date, settings: Settings) -> DeliveryResult`:
   - Set resend.api_key = settings.resend_api_key INSIDE the function (not at module level -- per RESEARCH.md pitfall 3)
   - Build params dict: from = f"BioIntelligence <{settings.sender_email}>", to = [settings.recipient_email], subject, html, text
   - Call _send_via_resend(params) which is the tenacity-wrapped function
   - On success: return DeliveryResult(date=target_date, email_id=response["id"], success=True)
   - On exception: log error, return DeliveryResult(date=target_date, success=False, error=str(e))

`_send_via_resend(params: dict) -> dict`:
   - Decorated with @retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=2, min=4, max=60), retry=retry_if_exception_type(Exception))
   - Calls resend.Emails.send(params)
   - Returns the response dict

Use structlog for logging send_email success/failure with email_id.
